**MQR 贡献分析（#29 示例）：**

```
Query: "如果要同时备考计算机二级 Python 和三级网络技术，需要分别掌握哪些编程和网络配置技能？"

MQR 拆解:
  子查询1: "计算机二级 Python 考试编程技能" → 检索到 Python 二级大纲
  子查询2: "三级网络技术考试网络配置技能" → 检索到三级网络大纲

无 MQR 的情况:
  原始 embedding 被 "Python" 主导 → 只检索到 Python 大纲，缺失网络技术内容
```

## 4. MQR 技术实现要点

### 4.1 查询拆解

| 组件 | 说明 |
|------|------|
| `is_cross_doc_query()` | 启发式检测：连接词、多问号、综合模式 |
| `query_decompose()` | FC 结构化输出 (mimo-v2.5) + 5 次重试 |
| `_rule_based_decompose()` | FC 失败时的规则兜底：连接词拆分、综合模式拆分、多问号拆分 |

### 4.2 检索融合

| 组件 | 说明 |
|------|------|
| `multi_search()` | 每个子查询独立执行向量+BM25+RRF，全局去重 |
| `skip_reranker=True` | 跨文档查询跳过 reranker（reranker 用原始查询打分，偏向主话题） |
| `_diversify_by_source()` | 按 source 轮询选取，保证每个子查询对应的文档都有机会出现 |

### 4.3 路由覆盖

| 组件 | 说明 |
|------|------|
| 跨文档路由覆盖 | `chain.py` 中 `is_cross_doc_query()` + 主题关键词匹配 → 强制 `general → rag` |

## 5. 已知问题与改进方向

### 5.1 失败用例分析

| # | 类型 | 问题 | 根因 | 改进方向 |
|---|------|------|------|---------|
| 13 | conceptual | langchain-community vs langchain 主包 | 知识库缺少直接对比内容，LLM 只能推测 | 补充模块化管理文档中的明确对比段落 |
| 17 | conceptual | 小优教育助手小学 vs 中学表述 | 知识库仅描述小学策略，无中学对比 | 补充中学教学策略到提示词文档 |

### 5.2 改进计划

| 优先级 | 方向 | 预期收益 |
|--------|------|---------|
| P1 | 知识库补充（conceptual 类缺失内容） | conceptual 通过率 50% → 75%+ |
| P2 | MQR 子查询数量自适应（当前固定 max=3） | 更复杂的跨文档查询支持 |
| P2 | Reranker 支持多查询打分（而非仅原始查询） | 不跳过 reranker 也能保持跨文档覆盖 |
| P3 | 前端 MQR 子查询展示 | 用户可见查询拆解过程 |
